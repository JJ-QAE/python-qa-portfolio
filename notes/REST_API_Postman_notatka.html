4 metody HTTP

GET

Pobierz dane

Odpowiednik: odczyt pliku. Nigdy nie ma body.

GET /users

GET /users/5
POST

Utwórz nowy zasób

Wymaga body JSON i headera Content-Type.

POST /users

+ body: {"{"}"name": "Anna"{"}"}
PUT

Zaktualizuj zasób

Nadpisuje cały obiekt. Wymaga body JSON.

PUT /users/5

+ body: cały obiekt
DELETE

Usuń zasób

Brak body w requeście. Id podajesz w URL.

DELETE /users/5
Ważne: GET nigdy nie ma body — dane przekazuje się tylko przez URL (query params).

Kody statusu

2xx — Sukces
200 OK — zadziałało
201 Created — utworzono
204 No Content — ok, brak treści
4xx — Twój błąd
400 Bad Request — zły format
401 Unauthorized — brak auth
404 Not Found — nie ma zasobu
5xx — Błąd serwera
500 Internal Server Error
502 Bad Gateway
503 Service Unavailable
4xx = sprawdź swój request. 5xx = problem po stronie serwera, nie twój błąd.

Autoryzacja i headery

Header Wartość Kiedy
x-api-key twój_klucz autoryzacja przez API key
Authorization Bearer eyJhbGci... autoryzacja przez token
Content-Type application/json gdy wysyłasz body JSON (POST, PUT)
Bezpieczeństwo: Klucze API to dane wrażliwe. Nie wklejaj ich do czatu, maila, Slacka ani na GitHuba.

Jeśli zmienne nie działają (401/400) — sprawdź czy środowisko jest wybrane w prawym górnym rogu Postmana.

Testy automatyczne — zakładka Scripts

Zakładka Scripts → Post-response — odpala się automatycznie po każdym Send.

// Sprawdzanie statusu
pm.test("Status powinien być 200", function () {
    pm.response.to.have.status(200);
});

// Sprawdzanie czy response zawiera pole
pm.test("Response zawiera id", function () {
    const json = pm.response.json();
    pm.expect(json[0]).to.have.property("id");
});
Wyniki widać w zakładce Test Results — np. 2/2 = 2 testy zaliczone na 2 możliwe.

Ściąga — co sprawdzić gdy coś nie działa

  • 01Zły status → sprawdź metodę i URL
  • 02401 → sprawdź autoryzację i czy środowisko jest wybrane w prawym górnym rogu
  • 03400 → sprawdź body — GET nie może mieć body, POST/PUT musi mieć Content-Type: application/json
  • 045xx → problem po stronie serwera, spróbuj później — to nie twój błąd
  • 05Zmienne nie działają → sprawdź czy środowisko jest wybrane w prawym górnym rogu Postmana
