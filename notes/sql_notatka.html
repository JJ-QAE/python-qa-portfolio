SELECT
co chcę zobaczyć
→
FROM
skąd (która tabela)
→
WHERE
warunek (opcjonalnie)
→
ORDER BY
sortowanie (opcjonalnie)
Zawsze w tej kolejności. SQL nie działa jeśli przestawisz kolejność.
SELECT — wybieranie kolumn
Wszystkie kolumny
SELECT * FROM pracownicy;
Gwiazdka * oznacza "wszystko".
Wybrane kolumny
SELECT imie, pensja FROM pracownicy;
Nazwy kolumn po przecinku — tylko te się pojawią.
WHERE — filtrowanie
Tekst
SELECT * FROM pracownicy
WHERE dzial = 'IT';
Tekst zawsze w apostrofach '...'
Liczba
SELECT * FROM pracownicy
WHERE pensja > 5000;
Liczby bez apostrofów. Działa też <, =, !=
AND — oba warunki
WHERE dzial = 'IT'
AND miasto = 'Warszawa'
Zawęża wyniki
OR — jeden z warunków
WHERE dzial = 'IT'
OR miasto = 'Warszawa'
Rozszerza wyniki
ORDER BY — sortowanie
Malejąco (od największej)
ORDER BY pensja DESC
DESC = descending (malejąco)
Rosnąco (od najmniejszej)
ORDER BY pensja ASC
ASC = ascending (domyślne)
Funkcje — COUNT, SUM, AVG
Funkcje idą po SELECT — zamiast surowych danych dostajesz obliczoną wartość.
Ile wierszy
SELECT COUNT(*) FROM pracownicy;
Liczy wszystkie wiersze
Ile spełnia warunek
SELECT COUNT(*) FROM pracownicy
WHERE dzial = 'IT';
COUNT + WHERE = potężna kombinacja
Suma
SELECT SUM(pensja) FROM pracownicy;
Dodaje wszystkie wartości w kolumnie
Średnia
SELECT AVG(pensja) FROM pracownicy;
Średnia arytmetyczna kolumny
GROUP BY — grupowanie
Wynik osobno dla każdej grupy
SELECT dzial, AVG(pensja)
FROM pracownicy
GROUP BY dzial;
Średnia pensja osobno dla każdego działu.
Liczba osób w każdym dziale
SELECT dzial, COUNT(*)
FROM pracownicy
GROUP BY dzial;
Uwaga: jeśli używasz GROUP BY, po SELECT możesz mieć tylko kolumny z GROUP BY i funkcje (COUNT, SUM, AVG). Nie możesz wyświetlić np. imie razem z GROUP BY dzial.
JOIN — łączenie tabel
JOIN skleja dwie tabele w jedną na podstawie wspólnej kolumny (np. id).
Podstawowy JOIN
SELECT p.imie, z.produkt
FROM pracownicy p
JOIN zamowienia z ON p.id = z.pracownik_id;
p i z to aliasy — skróty nazw tabel.
p.imie — kropka oznacza "z której tabeli biorę kolumnę".
ON — warunek połączenia, gdzie numery się zgadzają.
JOIN + WHERE
SELECT p.imie, z.produkt, z.kwota
FROM pracownicy p
JOIN zamowienia z ON p.id = z.pracownik_id
WHERE p.dzial = 'IT';
JOIN idzie przed WHERE — zawsze.
Kolejność z JOIN: SELECT → FROM → JOIN → ON → WHERE → ORDER BY
